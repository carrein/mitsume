# Notes Canvas — Research Findings
Created: 2026-07-10 (uberthink Phase 2; three web-research agents + local node_modules verification)

## Yjs stack

- **Versions:** `yjs@^13.6.31` (v14 is rc-only — do not use), `y-indexeddb@^9.0.12`, `idb@^8` (~1.2 KB brotli). yjs ≈ 28 KB gzip; Metro on SDK 56 resolves its exports map natively — no config needed. Watch for the "Yjs was already imported" console warning (dual-instance hazard; breaks instanceof/UndoManager origins).
- **Hermes:** yjs is pure JS. `TextEncoder`/`TextDecoder` already polyfilled in `src/polyfills`. Add a guarded `crypto.getRandomValues` polyfill from `expo-crypto` (lib0 falls back to Math.random otherwise — works, but clientID hygiene).
- **y-indexeddb:** web-only (`indexedDB` global). One `IndexeddbPersistence(name, doc)` per doc; gate first render on `whenSynced`; `destroy()` ≠ `clearData()`. Native V1: bare in-memory Y.Doc, no provider (data lost on restart — accepted for V1). No cross-tab live sync (fine, note it).
- **Doc shape (decided):** ONE `Y.Doc` → `getMap('canvases')`: canvasId → `Y.Map{icon, createdAt, items: Y.Map}`; items keyed by itemId → `Y.Map{blobHash, x, y, w, h, z}`. Nested Y.Maps (not plain object values) so a move rewrites only x/y. Y.Map over Y.Array (order comes from z/createdAt, O(1) delete). y-sweet is per-doc (auth/persistence/connection) → eventual sync likely doc-per-canvas + index doc; keep ALL doc access behind a store module so the split is mechanical. y-sweet itself is v0.9.1 (Sept 2025, pre-1.0) — re-validate health before the sync milestone.
- **UndoManager:** scoping to the current canvas's `items` Y.Map covers the whole subtree. `trackedOrigins: new Set([UI_ORIGIN])` (module-level exported const; matched by reference) — then EVERY undoable mutation must go through `doc.transact(fn, UI_ORIGIN)`; provider loads arrive with other origins → not undoable (desired). Recreate the UndoManager on canvas switch. `stopCapturing()` before each new gesture so quick successive ops don't merge (default captureTimeout 500ms).
- **Drag pattern:** never write Yjs per frame. Ephemeral local state during gesture, ONE transact on drop → one undo step, no y-indexeddb log bloat. (Multiplayer later: mid-drag goes to Awareness, same commit-on-drop.) Beware post-drop flicker: rendered position prefers ephemeral state while `isDragging`.
- **Blob store:** separate IndexedDB db `mitsume-blobs`, one store, key = SHA-256 hex, value = the `Blob` itself (structured clone supports Blobs everywhere modern; never base64). `idb` wrapper. `crypto.subtle.digest` — requires secure context; `http://localhost` IS one (dev :8880 OK), https prod OK; plain-http tailnet origin would NOT be → startup assertion, not a JS fallback. Object URLs: refcounted module cache hash → {url, count}; revoke at 0.
- **React binding:** hand-roll `useYSnapshot` — `useSyncExternalStore` + `observeDeep` + cached `toJSON()` snapshot (cache invalidated in observer; NEVER call toJSON directly in getSnapshot — infinite loop). No maintained lib worth adopting (react-yjs frozen, zustand-yjs dead). React Compiler safe: useSyncExternalStore is the sanctioned external-store path; route all Yjs reads through the hook.
- **Tests:** store logic testable against bare `Y.Doc`s (no persistence) under `bun test`; `fake-indexeddb` only if we ever test persistence.

## Canvas mechanics (RN + RN Web)

- **Reanimated 4.3.1 + worklets 0.8.3 + RN 0.85:** compatible pair; web supported (pure-JS fallback). babel-preset-expo auto-adds `react-native-worklets/plugin` when installed — no babel.config.js needed (verified locally in node_modules). React Compiler officially compatible (`.get()/.set()` API; gestures auto-memoized) — `'use no memo'` NOT required. **V1 decision: skip reanimated entirely** — camera as plain React state (rAF-throttled) is sufficient at V1 item counts and removes a whole risk class; reanimated can be layered in later for 60fps polish.
- **gesture-handler 2.31 web:** PointerEvent-based, default implementation; Pan handles mouse reliably (`mouseButton`, `minDistance` to distinguish click vs drag). `GestureHandlerRootView` required at app root on all platforms incl. web (expo-router does NOT provide one — add to `_layout.tsx`).
- **Wheel events:** no RN/RNW `onWheel` prop — attach a DOM `'wheel'` listener to the View ref (on RNW 0.21 the host ref IS the HTMLElement; findNodeHandle is gone). `{ passive: false }` + `preventDefault()` (blocks page scroll AND browser pinch-zoom). `e.ctrlKey` on wheel = trackpad pinch (Chrome/FF/Edge) → treat as zoom too even though plain wheel already zooms.
- **Zoom-at-cursor:** `screen = world*scale + translate` ⇒ `newT = cursor − (cursor − oldT)·(newScale/oldScale)`; cursor coords viewport-relative (`clientX − rect.left`). Formula assumes top-left origin; **RN transforms are center-origin by default** → put `transformOrigin: 'top left'` (RN ≥0.73 + RNW 0.21 both support it — verified) on a 0×0 absolutely-positioned anchor layer (belt and suspenders), children absolutely placed at world coords, transform `[{translateX},{translateY},{scale}]`.
- **Cursor styles:** work at runtime on web (`grab`, `nwse-resize`...) but RN types only allow `'auto'|'pointer'` → small typed helper with a web-only cast.
- **Dot grid:** do what tldraw does — grid rendered SCREEN-FIXED (outside the transform), spacing `s = 32·scale`, offset `((t % s)+s)%s`; react-native-svg `<Defs><Pattern patternUnits="userSpaceOnUse">` + full-viewport `<Rect fill="url(#dots)">` works on web AND Android (RNSVG web emits real DOM SVG). Fade/step the grid when `s` gets small (double gridSize or fade opacity below ~16px spacing) so far zoom-out never rasterizes thousands of dots. Never put a giant rect inside the scale transform (blur + unbounded repaint).
- **Images:** expo-image (already a dep) renders a plain `<img>` on web — `blob:` object URLs and `data:` URIs pass straight through IF `source.headers` is unset. Same component takes `file://` on Android later (Glide + disk cache). Use expo-image everywhere.
- **Paste:** `document.addEventListener('paste')` — `e.clipboardData.files` first (Safari's surface), fall back to `items` loop with `getAsFile()` (Chromium screenshots). No permission prompts ever (paste keystroke IS the grant). Bail when target is INPUT/TEXTAREA/contentEditable. Safari is the variance king — manually test screenshot-paste there. Dimensions: `createImageBitmap(blob)` → width/height, `.close()` after; EXIF-safe in modern Safari.
- **Keyboard:** document-level `'keydown'` in a `.web.ts` hook; skip editable targets; Delete/Backspace → delete selection; mod+Z / mod+shift+Z → undo/redo. No universal RN API worth using.
- **Prior art:** no maintained RN/RNW infinite-canvas lib exists — building a thin camera layer is the normal move. Steal patterns: tldraw (screen-fixed grid, camera {x,y,z}; custom license — patterns only, not code), react-native-zoom-reanimated (pinch focal-point math for the Android phase), react-native-free-canvas (Gesture.Simultaneous(Pinch, Pan) composition).

## Backend (round 2 — server as source of truth; researched 2026-07-10)

- **y-sweet is dead.** Jamsocket founders joined Modal 2025-07-10; jamsocket.com says "shutting down" (HTTP 402); docs.jamsocket.com NXDOMAIN; last maintainer commit 2025-09-29; last release v0.9.1 (2025-09-16); bugfix PRs unanswered; ghcr image amd64-only and frozen. The spec's choice did not survive.
- **MinIO OSS is dead.** Console gutted May 2025 → last release Oct 2025 (that CVE fix never imaged) → "NO LONGER MAINTAINED" Feb 2026 → repo archived 2026-04-25. Docker Hub/quay images frozen at RELEASE.2025-09-07. Community verdict: migrate.
- **Sync replacement (chosen): Hocuspocus v4** — `@hocuspocus/server` 4.3.0 (2026-06-18), MIT, Tiptap-backed, commits current, v4 stable May 2026, runs on Node/Bun/Deno/CF Workers, authless by default (`onAuthenticate` optional), provider is editor-agnostic. Persistence: `extension-sqlite` (better-sqlite3) or `extension-database` (generic async fetch/store hooks — pair with `bun:sqlite` for a zero-native-dep Bun image). No official Docker image — we own a ~20-line entry + Dockerfile. `HocuspocusProvider` takes an explicit ws URL → subpath-clean behind Caddy `handle_path /sync/*` (websocket upgrade is transparent).
- **Blob replacement (chosen): caddy-webdav container** — xcaddy build (`caddy:2.11-builder` + `github.com/mholt/caddy-webdav`, plugin alive, maintained by Caddy's author). Plain directory, filename = SHA-256: PUT via webdav, GET/HEAD via file_server, DELETE enabled (webdav) for the delete-removes-bytes requirement. Caddy has NO default body limit (add `request_body max_size`); host Caddy injects `Cache-Control: public, max-age=31536000, immutable` on GET/HEAD (deferred `>` header rewrite) and must not cache-header 404s. Anonymous PUT alternatives all lose: Garage can't anon-PUT until v3 (PR in review), SeaweedFS wants ~2GB RAM, rclone-serve-s3 keeps metadata in memory, nginx-dav adds a second proxy tech. **Caveat:** x/net/webdav PUT writes in place — a dropped connection can leave a truncated file under a valid hash; mitigate client-side with HEAD Content-Length verification after PUT.
- **Client blob cache/queue canon:** NO service worker, NO Background Sync (Chromium-only; Mozilla/WebKit negative). One IDB store `blobs` (key = hash, value = Blob + mime + `uploaded` flag) — the pending queue IS `uploaded:false`, so blob and queue can't disagree. Foreground drain loop: triggers = app start, `online` event (hint only, never a gate), visibilitychange→visible, post-paste, slow interval while pending; per-item exponential backoff + jitter; fetch outcome is the truth. Dedup: HEAD (`cache:'no-store'`) → 404 ⇒ PUT → verify HEAD. Read path: IDB hit → object URL; miss → GET (browser HTTP cache absorbs repeats thanks to `immutable`) → write back to IDB as the offline set. `navigator.storage.persist()`: call once, log, design as if denied (Chrome grants silently by heuristics only). idb-keyval 6.3.0 / idb 8.0.3 both fine; hand-roll the queue (~100 lines), skip Workbox/Dexie.
- **Provider composition canon:** y-indexeddb + network provider on one Y.Doc is the blessed meshable pattern; sync is state-vector-based (no full re-downloads). Gate first paint on y-indexeddb `synced`. **Seeding gotcha:** never construct "initial content" independently on two clients (sequence CRDTs duplicate); the default canvas must use a FIXED map key ('default') and be created only after local sync — Y.Map same-key sets converge, but concurrent creations with fresh nested maps can drop one side's early items (accepted single-user risk, noted).
- **Secure context:** prod origin is HTTPS via tailnet certs (host Caddy + tailscaled.sock) and dev is localhost ⇒ `crypto.subtle` available in both. `@noble/hashes` fallback only if a plain-http origin ever matters (don't add now).
- **Host conventions (from the real compose):** one host stack; per-service hardening (`no-new-privileges`, `cap_drop: ALL` + minimal adds, mem limits, healthchecks, flame + watchtower labels), MITSUME_* env vars on the caddy service, 127.0.0.1-bound port publishes, ZPOOL bind mounts. New services join the EXISTING mitsume site block (same origin) — repo ships merge-ready fragments, Deploy.md-style.
- **Hocuspocus ops notes:** our entry should expose an HTTP /health route (we own the server file); `onStoreDocument` is debounced (~2s default, unverified — check at build time). Native RN has no indexedDB → native runs provider-only (WebSocket exists on RN), no local persistence, V1.
